## 🚀 Deployment & Hosting Guides

### 1. Shared Hosting Deployment Guide
Shared hosting environments are highly constrained (no root terminal access, no Redis daemon, no Supervisor daemon, no daemonized WebSockets). This application is architected to run perfectly within these constraints.

#### Structure & Uploading
1. **Prepare Structure**: Move the contents of the `public/` directory into the server's `public_html/` folder (or relevant root document folder) to keep the app core private.
2. **Path Configuration**:
   - In `public_html/index.php`, update the paths to point to the parent directory:
     ```php
     require __DIR__.'/../bootstrap/app.php';
     ```
3. **Storage Symlink**:
   - Since you cannot run `php artisan storage:link` on some shared hostings, create a route in `routes/web.php` to generate it programmatically, then visit it:
     ```php
     Route::get('/symlink', function () {
         symlink(storage_path('app/public'), public_path('storage'));
         return 'Storage link created!';
     });
     ```
   - Alternatively, configure a symlink via cPanel's File Manager or Terminal.

#### Database & Environment
1. Set up a MySQL database using cPanel's MySQL Database Wizard.
2. Edit the `.env` file in the parent folder, defining the correct database credentials:
   ```env
   APP_ENV=production
   APP_DEBUG=false
   DB_CONNECTION=mysql
   QUEUE_CONNECTION=database
   ```
3. Secure the `.env` file in the parent folder by adding this rule to your `.htaccess`:
   ```apache
   <Files .env>
   Order allow,deny
   Deny from all
   </Files>
   ```

#### Queues & Scheduler via cPanel Crons
Because shared hosting lacks Supervisor to daemonize `queue:work` workers:
1. **Scheduler Cron**: Set up a cron job in cPanel to run every minute:
   ```bash
   * * * * * cd /home/username/project && php artisan schedule:run >> /dev/null 2>&1
   ```
2. **Queue Runner Cron**: To process broadcast queue messages without keeping a terminal open, set up a cron job in cPanel running every 2-5 minutes to consume pending jobs:
   ```bash
   */2 * * * * cd /home/username/project && php artisan queue:work --stop-when-empty >> /dev/null 2>&1
   ```
   *Note: `--stop-when-empty` makes the worker exit cleanly once the queue is empty, preventing persistent processes that shared hostings might terminate.*

### 2. VPS Hosting Deployment Guide (Ubuntu / Nginx / systemd / Supervisor)
VPS hosting environments give you full root control, allowing you to use high-performance daemons like Redis, Supervisor, and customized Webhook rules.

#### Nginx Virtual Host Setup
Create an Nginx configuration file (`/etc/nginx/sites-available/whatsapp-saas`):
```nginx
server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/whatsapp-saas/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;
    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.3-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
```
Enable the site and obtain an SSL certificate via Certbot (Let's Encrypt):
```bash
sudo ln -s /etc/nginx/sites-available/whatsapp-saas /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
sudo certbot --nginx -d yourdomain.com
```

#### Production Queues via Supervisor
Supervisor ensures that your queue worker runs continuously in the background, auto-restarting if it crashes.

1. Install Supervisor:
   ```bash
   sudo apt-get install supervisor
   ```
2. Create the configuration file (`/etc/supervisor/conf.d/whatsapp-worker.conf`):
   ```ini
   [program:whatsapp-worker]
   process_name=%(program_name)s_%(process_num)02d
   command=php /var/www/whatsapp-saas/artisan queue:work --sleep=3 --tries=3 --max-time=3600
   autostart=true
   autorestart=true
   stopasgroup=true
   killasgroup=true
   user=www-data
   numprocs=2
   redirect_stderr=true
   stdout_logfile=/var/www/whatsapp-saas/storage/logs/worker.log
   stopwaitsecs=3600
   ```
3. Load and start the worker daemon:
   ```bash
   sudo supervisorctl reread
   sudo supervisorctl add
   sudo supervisorctl start whatsapp-worker:*
   ```

#### Cron Scheduler Setup
Add the scheduler to the root system cron table (`/etc/crontab` or `crontab -e -u www-data`):
```bash
* * * * * www-data php /var/www/whatsapp-saas/artisan schedule:run >> /dev/null 2>&1
```

#### Webhook Permissions & Firewall Settings
Because Meta (Facebook) Graph servers must be able to hit your webhook endpoint securely:
1. Make sure your Nginx firewall allows incoming traffic on ports `80` and `443` (UFW configured):
   ```bash
   sudo ufw allow 'Nginx Full'
   ```
2. Ensure Meta verification requests don't hit rate limit blocks in Laravel:
   - Configure Webhook route namespaces to bypass strict anti-scraping middlewares.
   - Set up an proxy layer if your VPS is behind Cloudflare (make sure Cloudflare's SSL mode is set to **Full** or **Full (strict)** so Meta verification payloads pass without SSL challenge loops).
